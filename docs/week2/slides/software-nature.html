Eigenschaften von Software

Marcel Lüthi
Departement Mathematik und Informatik

Das Produkt "Software"

  • Ingenieure entwickeln Produkte
Software ist irgendwie anders!

Eigenschaft 1: Formbarkeit

  • Software ist formbar
    (engl. malleable)
  • Das Produkt lässt sich einfach verändern
  • Keine teure Änderung der Produktionsanlagen nötig
Aber: Das Design ist starr!

Gefahr der Formbarkeit

  • Software wird verändert ohne das Design anzupassen.
    • Produkt und Design divergieren

Eigenschaft 2: Immaterialität

  • Software ist immateriell

Traditionelles Produkt
  • Herstellung grösster Kostenfaktor.
  • Fokus: Reibungslose Produktion garantieren
    • Qualitätskriterien in Produktion
Software
  • Herstellung ist trivial (kopieren)
  • Fokus: Design und Implementation
    • Qualitätskriterien in Entwicklungsprozess

Physik als Leitfaden

Physik hilft uns Produkte zu entwickeln:
  • definiert Lösungsraum
  • hilft uns Eigenschaften abzuleiten
  • hilft uns Anforderungen zu testen

Eigenschaft 3: Software untersteht nicht den Naturgesetzen!

  • Software hat wenig verlässliche Strukturen
  • Logische Folgerungen / Ableiten von Eigenschaften sind schwierig
    • Erschwert sowohl Entwicklung als auch die Verifikation.
Alle logischen Strukturen müssen designed und dokumentiert werden.

Zusammenfassung

  • Software lässt sich leicht ändern - das Design aber nicht
  • Die Qualität der Software wird im Entwicklungsprozess erreicht
  • Verlässliche Strukturen kommen nicht von aussen, sondern müssen explizit designed werden
Um gute Software zu bauen brauchen wir Disziplin und gute Prozesse
